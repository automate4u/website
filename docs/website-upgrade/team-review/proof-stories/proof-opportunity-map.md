# Proof Opportunity Map

This file turns proof examples into sales opportunities the team can pursue deliberately.

The goal is not to publish every example immediately. The goal is to know which opportunities would create the strongest future case studies and which examples are worth turning into demos, pilots, or sales conversations.

## Proof Type Distinction

Keep these categories separate:

- **Automate4U client proof:** work the company has delivered for Automate4U clients. This can become a sanitized case study if approved.
- **Founder/operator experience proof:** work an Automate4U founder or team member personally delivered in prior roles or ventures. This can support credibility, but should be described as founder/team experience unless it was an Automate4U client project.
- **Representative workflow example:** a realistic pattern Automate4U can pursue or demonstrate. This should not be presented as completed client work until backed by a real approved engagement.
- **Demo proof:** a sample workflow using fake data to show how the system would work. This should be labeled as a demo or illustrative workflow.

## How To Use This Map

For each opportunity:

1. Build or refine a small demo.
2. Use it in sales conversations with the matching buyer type.
3. Sell a bounded pilot.
4. Measure the result.
5. Convert the work into a sanitized case study or representative proof asset.

## Capability Proof Lanes From Michael's Background

These are credible because they align with prior work in production AI agents, RAG, integrations, SaaS, ecommerce, observability, and compliance-sensitive systems.

- Production LLM agent systems: intake, qualification, scheduling, follow-up, tool-calling, retries, fallbacks.
- RAG and document intelligence: ingestion, chunking, embeddings, retrieval, answer grounding, evaluations.
- Enterprise integrations: Google Workspace, Microsoft, Zoom, Slack, Workday-like HR systems, Okta/SSO, SendGrid-style messaging.
- Reliability and observability: structured logs, tracing, dashboards, alerting, runbooks, incident response.
- Multi-tenant SaaS: permissions, admin tools, user roles, analytics, deployment confidence, CI/CD.
- Ecommerce and marketplace operations: onboarding, catalog, inventory, orders, payments, notifications, internal operations tooling.
- Search and discovery: text search, filters, location, availability, relevance tuning, caching.
- Sensitive data workflows: access control, audit logs, data integrity, careful escalation, compliance-minded delivery.

Use these lanes as **founder/operator experience proof**, not as Automate4U case studies unless the work happened under Automate4U and is approved for public use.

## Proof Story Priority

Prioritize proof stories that satisfy at least three of these:

- The buyer immediately understands the pain.
- The workflow can be shown in a demo.
- The result can be measured.
- The automation connects multiple systems.
- Human control is clear.
- The story supports a repeatable offer.
- The story can lead to a paid pilot in 30 days or less.
