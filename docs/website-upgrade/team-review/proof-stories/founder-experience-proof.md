# Founder Experience Proof

Use this file to decide how Automate4U should use founder and team experience as website credibility.

The site should not rely only on client case studies. A small expert team can also build trust by showing that the people behind the company have personally delivered production systems, AI agents, integrations, ecommerce platforms, reliability work, and sensitive workflows before.

The important rule: **do not blur founder experience, Automate4U client work, and representative examples.**

## Proof Categories

### Automate4U Client Proof

Use when the work was delivered by Automate4U for a client.

Possible labels:

- Anonymous case study.
- Sanitized client story.
- Client workflow example.
- Representative pattern based on Automate4U client work.

### Founder / Operator Experience Proof

Use when the work was done by Michael or another team member in a prior role, venture, or contract.

Possible labels:

- Founder experience.
- Engineering leadership experience.
- Built by a team with production experience in...
- Our engineering leadership has built...
- Prior experience includes...

This is real credibility, but it should not be written as if Automate4U delivered it for an Automate4U client.

### Representative Workflow Example

Use when the example is realistic and aligned with team capability, but not yet tied to an approved client story.

Possible labels:

- Representative workflow example.
- Example implementation pattern.
- Illustrative workflow using sample data.
- Demo scenario.

## Candidate Founder Experience Claims

Michael has confirmed that the resume metrics can be used publicly if helpful. Still, use exact metrics selectively: they should strengthen buyer trust, not make the page feel like a resume or an inflated pitch.

## When To Say Founder, Leadership, Or Team

Use the wording based on context:

- **Founder experience** works best on About pages, founder bios, team credibility sections, investor/partner conversations, and deeper credibility pages. It is personal and transparent.
- **Engineering leadership** works best on service and capability pages when the buyer needs confidence in technical execution without making the copy about one person.
- **Our team has experience** works best on the homepage, industry pages, and general sales pages because it keeps the focus on Automate4U as the partner.
- **Automate4U has built** should only be used when the work was delivered through Automate4U and can be approved as client proof.

Practical examples:

- Homepage: "Built by a team with production experience across AI agents, integrations, ecommerce, SaaS, and operational software."
- About Team: "Founder experience includes production AI agents, RAG systems, enterprise integrations, ecommerce operations, and reliability work."
- AI Agents page: "Our engineering leadership designs agents around tool-calling, validation, retries, fallbacks, and human escalation."
- Case study page: "Automate4U built..." only when the story is an approved Automate4U client engagement.

## Prior Company And Client Naming

Default public approach:

- Use broad credibility labels such as "government agencies," "Fortune 500 environments," "SMBs," "marketplaces," "ecommerce operators," "workplace platforms," and "sensitive-data workflows."
- Avoid long name-dropping lists on primary sales pages.
- Use named prior companies only when they materially improve trust and the context is accurate.

Potential named examples available from Michael's background or prior ecosystem exposure include companies and platforms such as Google, Microsoft, Slack, Okta, Splunk, Upwork, Shipt, Webflow, and US government-related work. Use these carefully:

- On integration pages, it is fine to show many platforms as examples of systems we can connect with.
- On About pages, named prior roles can be useful if they help buyers understand depth.
- On service pages, prefer buyer-relevant capability language over name lists.

## Website Copy Patterns

Good:

- "Our engineering leadership has built production AI agent systems, RAG pipelines, enterprise integrations, ecommerce operations tooling, and reliability infrastructure."
- "That background shapes how we build: controlled pilots, grounded answers, system actions, monitoring, and human review."
- "Representative examples on this site show the kinds of workflows we can design; approved client stories are labeled separately."

Avoid:

- "Automate4U has delivered all of these examples" unless true and approved.
- "We guarantee production-grade AI."
- "Enterprise-level compliance" without specific certification.
- "Our AI replaces staff."
- Unverified metrics.
