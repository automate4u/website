## Before State

Order and payment events created operational edge cases: failed payments, refunds, disputes, delayed status updates, duplicate events, customer questions, and staff follow-up.

Without reliable event handling, support teams had to manually check order state and payment records.

## What The Automation Did

- Processed payment and order events.
- Used retries and idempotency to avoid duplicate actions.
- Updated order status.
- Triggered customer or internal notifications.
- Flagged failed payments, refunds, disputes, and edge cases.
- Created support tasks where manual review was needed.
- Gave staff a clearer operational view of order state.
