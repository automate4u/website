# Asset Redaction Checklist

Use this before any screenshot, transcript, dashboard, diagram, or video becomes public.

## Always Remove

- Client name, logo, brand colors, or domain.
- Customer names.
- Staff names.
- Phone numbers.
- Email addresses.
- Addresses.
- Account numbers.
- Order numbers.
- Part numbers if identifying.
- Child, parent, patient, student, or client records.
- Prices, margins, discounts, invoices, or payment details.
- API keys, tokens, webhook URLs, internal IDs, or database IDs.
- Proprietary prompts, policies, scripts, or internal operating procedures.
- Exact system combinations if they could identify the client.

## Usually Generalize

- Geography.
- Location count.
- Software names.
- Exact monthly volume.
- Exact savings or revenue impact.
- Unique service categories.
- Timeline and launch dates.
- Sensitive escalation rules.

## Safe Asset Ideas

- Recreated workflow diagrams using fake labels.
- Mocked dashboard with representative data.
- Fake transcript showing tone and handoff logic.
- Before/after workflow map.
- Sanitized queue view with generic request types.
- Event log showing stages such as captured, classified, routed, approved, completed.
- KPI panel using fictional example values clearly labeled as illustrative.

## Public Asset Labeling

Use clear labels:

- "Representative workflow example"
- "Illustrative demo using sample data"
- "Sanitized workflow diagram"
- "Example event log"
- "No client data shown"

Avoid labels that imply a fake asset is a direct client screenshot.

## Final Review Questions

- Could a client, competitor, employee, or vendor identify the client from this?
- Does the asset reveal anything operationally sensitive?
- Does the asset make a claim the text does not support?
- Is the asset necessary, or could a recreated diagram communicate the idea more safely?
- Has the team approved this for the intended page?
