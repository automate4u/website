# Team Review Docs

This folder contains the docs that are easiest for the Automate4U team to review and act on.

The files use plain names because they are review documents, not numbered implementation steps.

## Review First

- `current-status-and-review-checklist.md`: simple done/todo checklist, priority pages to review, remaining risks, and next actions.
- `production-lead-routing-setup.md`: HubSpot, Resend, Calendly, and PostHog setup instructions.
- `proof-inventory-checklist.md`: what to gather for anonymous/sanitized proof and case studies.
- `services-and-how-we-build-taxonomy.md`: how Services, Industries, Solutions, and How We Build fit together.
- `project-decisions-summary.md`: the important decisions we should not lose.

## Strategy And Messaging

- `market-positioning-strategy.md`: positioning, AI Voice wedge, broader automation story.
- `site-architecture-and-offers.md`: navigation, offers, and page model.
- `cta-lead-routing-analytics-and-pricing.md`: CTA language, lead flow, analytics, scheduling, pricing guidance.
- `pricing-transparency-and-cost-model.md`: pricing ranges and variable cost logic.
- `creative-experience-direction.md`: visual direction, professional-services trust, and design guardrails.

## Team-Owned Actions

The team should provide or review:

- HubSpot private app token and object model decision.
- Resend account, verified sender/domain, and API key.
- Final Calendly booking link.
- PostHog project key if using PostHog.
- Real team/process images when available.
- Proof inventory, approved metrics, and NDA-safe case study permissions.
- Legal review for Privacy and Terms.
- Cofounder review of AI Voice before major flow changes.
- Final judgment on claims, tone, pricing ranges, and service positioning.

## When To Hand Back To Codex

Hand decisions back to Codex when:

- Production routing credentials are ready.
- HubSpot field names or object model are confirmed.
- Team approves or changes page positioning.
- Proof inventory is ready to turn into anonymous/sanitized case studies.
- Legal/privacy review requires copy changes.
